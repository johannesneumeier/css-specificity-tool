Information

The way CSS declarations are applied to rendered elements depends on each selector’s specificity. This refers to the calculated ‘strength’ of that selector. Generally speaking, more specific selectors should take precedence over more general selectors.

For instance: body { color: grey; } is a very general selector setting the color of all document text to grey. header nav { color: red; } is more specific. The same for #navigation { color: red; }. Both are more specific, either due to more precise elements or because of an ID selector that identifies one very element only.

By inputting your selectors in the above calculator you can find out which selector’s specificity is strongest.

Disclaimer
You can use this tool “as is” without any warranty or garanties of correctness.
  • The tool falsely calculates :not() selector pseudo attributes, which it should not do.
  • Currently there is no way of inputting or processing style="" inline selectors, i.e., this means the first numeric part of the specificity will never show as counted.
  • !important declarations always overwrite calculated specificity, be mindful when using it for this very reason.
